The central result

Nine agents beat B&H by +2.91pp. Three agents lose to the market. Composition matters more than count.

We compare configurations of 1, 3, 5, and 9 LLM agents for algorithmic trading on Bitcoin. Each agent receives real BTC market data at 8 decision points over 84 trading days and votes BUY, SELL, or HOLD according to its assigned personality prompt. The only experimental variable is the number and composition of agents voting on each decision.

Winning panel · 9 agents

Diverse biases, balanced signals

The 9-agent panel provides a 56%–33%–11% split (bullish–bearish–mixed). When the market declines (late March 2026), bearish agents vote SELL, reducing position size and limiting losses. During temporary rallies (April–May 2026), bullish agents vote BUY, capturing upside. Mixed agents moderate extremes, preventing domination by either faction.

§3 Methodology

Rigorous experimental design with real data and real LLM inference.

3.1 Market Data

We use BTC/USDT spot data from February 22 to June 15, 2026 (84 trading days). Bitcoin declined from ~$96,000 to ~$72,000 (−6.45%). Data sourced from Yahoo Finance (yfinance). 8 decision points spaced ~10 trading days apart. Each agent receives: current price, 1d/1m/3m returns, RSI(14), 5-period price range, and volume relative to 20-period average.

3.2 Agent Design

Nine agents, each with a unique English-language prompt. The prompts describe different trading philosophies (momentum, mean reversion, trend following, breakout, value, volatility aversion, range trading, volume confirmation, risk parity). All agents use the same base model (Gemma 7B) with different instructions — diversity comes from prompt design, not model architecture.

3.3 Voting Mechanism

BUY = +0.7, SELL = −0.7, HOLD = 0.0. The panel average determines position size (0–100% of capital). Unlike majority voting, this produces a continuous position signal that reflects the panel's conviction level.

3.4 Infrastructure

Gemma 7B (Q4, 5 GB) via Ollama on a standard MacBook. No GPU, zero API costs. 72 calls in ~45 minutes. Full reproducibility without external dependencies.

Abstract

A controlled experiment in multi-agent LLM trading.

We present an empirical comparison of multi-agent LLM configurations for algorithmic trading on Bitcoin, using 72 real inference calls to Gemma 7B via local inference (Ollama, zero API cost). We evaluate configurations of 1, 3, 5, and 9 autonomous agents across 8 decision points over 84 trading days of real BTC/USDT market data. Each agent receives identical market data and votes BUY, SELL, or HOLD according to its assigned personality prompt (Momentum, Mean Reversion, Trend Follower, Breakout, Value, Volatility Averse, Range, Volume, Risk Parity).

The 9-agent configuration (−3.54%) outperforms Buy & Hold (−6.45%) by +2.91 percentage points — a 45% reduction in losses. The 3-agent configuration (−7.15%) underperforms the market. Panel composition — not agent count — is the primary determinant of performance. The winning 9-agent panel (56% bullish, 33% bearish, 11% mixed) provides balanced signal coverage. The losing 3-agent panel polarizes votes into near-zero positions.

Analysis reveals three consistent behavioral clusters: three agents vote BUY in every decision, three rarely vote BUY, and three show conditional behavior. This stability validates prompt-based personality design. We release all prompts, code, and data for reproducibility.

§5 Discussion

Composition matters more than count. Implications for multi-agent LLM design.

5.1 Main Finding

Adding agents does not guarantee improvement. A 5th bullish agent to a bullish-heavy panel has less marginal benefit than a single contrarian agent. This aligns with ensemble learning theory: diversity among base learners is more important than the number of learners (Dietterich, 2000; Brown et al., 2005). In LLM agents, diversity comes from prompt design — a form of representation diversity where different prompts create different feature interpretations of the same input data.

5.2 Practical Design Principles

(1) Prioritize diversity over quantity. The marginal benefit of each agent depends on how much new diversity it brings. (2) Balance is measurable. The optimal panel (~55% bullish, ~35% bearish, ~10% mixed) can be calibrated through pre-experiment piloting. (3) Personality consistency enables reliable aggregation. The stability of agent behavior (Table 3) validates that LLMs can serve as reliable multi-agent components. (4) Avoid polarized panels. Equal opposing representation produces decision paralysis.

5.3 Limitations (7)

(1) Single bearish market regime (BTC −6.45%). (2) Only 8 decisions — insufficient for statistical significance. (3) Single model (Gemma 7B). (4) No transaction costs (slippage, commissions). (5) No inter-agent deliberation. (6) No currency risk modeling. (7) Limited personality space — 9 personalities don't span all possible strategies.

5.4 Future Work (7 directions)

(1) 50+ decisions across bull/bear/sideways regimes. (2) Cross-model comparison (Llama 3, Qwen, Claude). (3) Grid search over panel sizes (2–8 agents). (4) Performance-weighted voting. (5) Multi-agent debate architectures. (6) Transaction cost modeling. (7) Adaptive composition based on detected market regimes.
